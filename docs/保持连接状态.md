# 现状
- 用户输入 /chrome-start 命令后，会打开chrome并连接好
- 连接状态下，用户退出pi，会自动断开连接，不关闭chrome
- 连接状态下，用户 /new 时程序没有处理，所以会丢掉连接
- 连接状态下，用户输入 /chrome-stop 会关闭连接。

# 目标设计

用户执行一次 `/chrome-start` 后，Chrome 连接在 pi 的整个生命周期内持续有效。包括 `/new`、`/resume`、`/fork` 等操作后也能自动恢复连接，直到用户显式执行 `/chrome-stop`。

## 核心思路

用文件标志持久化用户的「连接意图」。`session_start` 时检查标志 + Chrome 是否可达，满足条件则自动重连。

## 标志文件

路径：`~/.pi/tmp/chrome-connection.json`

```json
{
  "mode": "local",
  "host": "127.0.0.1",
  "port": 9222,
  "timestamp": 1748232000
}
```

- `mode`: `"local"`（本机 Chrome）或 `"remote"`（SSH 端口转发到远程 Mac）
- `host` / `port`: 连接配置（远程模式下 `host` 为本地 SSH 隧道绑定的地址，通常为 `127.0.0.1`；重连时会用这些值调用 `configureConnection()` 恢复连接参数）
- `timestamp`: 写入时间，`session_start` 检查时超过 24h 则视为过期，静默清标志

> **注意**：远程模式下，标志文件只记录隧道本地端的地址和端口。自动重连要求 SSH 隧道仍然存活。如果隧道已断开，重连会静默失败并清除标志文件，不会打扰用户。

## 各场景行为

### `/chrome-start`
- 如果有 Chrome 在运行 → 直接连接
- 如果没有 Chrome 在运行 → 启动 Chrome，然后连接
- 连接成功 → 写标志文件
- 连接/启动失败 → 通知用户 + 清除标志文件（避免后续自动重连失败状态）

### `/chrome-stop`
- 断开连接，**不关闭 Chrome**（用户可能继续用 Chrome 浏览网页）
- 删除标志文件
- ⚠️ **Breaking Change**：本地模式下旧行为会 kill Chrome，新行为统一为只断开不关闭

### `session_shutdown` 统一处理

所有场景共用一个 `session_shutdown` 处理器，通过 `event.reason` 区分行为：

| reason | 场景 | 断开连接 | 关闭 Chrome | 标志文件 |
|--------|------|---------|------------|----------|
| `quit` | 退出 pi | ✅ | ❌ | 清除 |
| `new` | /new | ✅ | ❌ | 保留 |
| `resume` | /resume | ✅ | ❌ | 保留 |
| `fork` | /fork | ✅ | ❌ | 保留 |
| `reload` | /reload | ✅ | ❌ | 保留 |

逻辑：
```
断开连接（不关 Chrome）
if (reason === "quit") 清除标志文件
else 保留标志文件（新 session 会自动重连）
```

### `session_start` 自动重连

当 reason 为 `new` | `resume` | `fork` | `reload` 时（不包括 `startup`——用户从 shell 启动 pi 时不需要自动重连）：
- 检查标志文件是否存在
- 存在 → 检查 `timestamp` 是否超过 24h
  - 过期 → 静默清标志，不重连
  - 未过期 → **先从标志文件读取 `mode`/`host`/`port`，调用 `browser.configureConnection()` 恢复连接参数**（关键：`session_shutdown` 会销毁整个扩展运行时，模块级变量 `debugHost`/`debugPort`/`connectionMode` 会回到默认值）
    - 检查 Chrome 是否可达
      - 可达 → 自动重连，注册工具，用 `ctx.ui.notify()` 发一条低干扰通知（如 "✅ Chrome 已自动重连"）
        - `session_start` handler 签名为 `ExtensionHandler<SessionStartEvent>`，接收 `(event, ctx: ExtensionContext)`，`ExtensionContext` 包含 `ui: ExtensionUIContext`，可直接调用 `ctx.ui.notify()`
      - 不可达 → 删除标志文件，用 `ctx.ui.notify()` 低干扰通知用户（如 "ℹ️ Chrome 已不可达，未自动重连"）
- 不存在 → 什么都不做

## pi 事件钩子

```
session_start { reason: "new" | "resume" | "fork" | "reload" }（startup 不重连）
  → 读标志文件 → 未过期 → configureConnection() → Chrome 可达 → 静默重连

session_shutdown { reason: "quit" | "new" | "resume" | "fork" | "reload" }
  → 先设 isShuttingDown 标志（确保 onDisconnected 在断开前已被屏蔽）
  → 断开连接
  → reason=="quit" → 清标志
  → 其他 → 保留标志
```

## 实现步骤

### 1. 新增 `core/connection-state.ts`
- `readConnectionState(): ConnectionState | null` — 读标志文件
- `writeConnectionState(state)` — 写标志文件（先用 `mkdirSync(dir, { recursive: true })` 确保 `~/.pi/tmp/` 目录存在）
- `clearConnectionState()` — 删标志文件
- `isChromeReachable(state): Promise<boolean>` — 检查 Chrome 是否可用（复用 `browser.isChromeRunning()` 或直接请求 `http://{host}:{port}/json/version`）

### 2. 提取共享连接建立逻辑

`/chrome-start` 和 `session_start` 自动重连需要执行相同的步骤：
- 注册 `onDisconnected` 回调
- `consoleBuffer.startListening()`
- `registerTools()` + `setActiveTools()`
- 发通知

提取为共享函数 `establishConnection(pi, ctx, consoleBuffer)`，两处调用同一函数，避免逻辑分叉。
- `/chrome-start` 的 handler 接收 `ctx: ExtensionCommandContext`（extends `ExtensionContext`）
- `session_start` handler 接收 `ctx: ExtensionContext`
- 两者都有 `ctx.ui.notify()`，可直接共用同一通知方式
- 注意：断线回调 `onDisconnected` 没有 `ctx`，仍用 `pi.sendMessage()` 发断线通知

### 3. 修改 `index.ts`
- `session_start` 处理器中增加自动重连逻辑：
  - 读取标志文件 → `configureConnection()` → `isChromeReachable()` → `establishConnection(pi, ctx, consoleBuffer)`
  - 用 `ctx.ui.notify()` 发通知
  - 加 `isReconnecting` 标志防止与用户同时执行 `/chrome-start` 的并发竞态
- `session_shutdown` 处理器中：
  - 屏蔽 `onDisconnected` 回调（先清 `toolNames` 或设 `isShuttingDown` 标志）
  - 断开连接
  - `reason === "quit"` 时清标志，否则保留
- `/chrome-start`：成功后调用 `writeConnectionState()`，失败时调用 `clearConnectionState()`
- `/chrome-stop`：改为只断开 + 清标志，**不 kill Chrome**（本地/远程统一行为，Breaking Change）
- 解决 `onDisconnected` 与 `session_shutdown` 的冲突：shutdown 断开前先屏蔽回调，避免发误导通知

### 4. 并发保护

添加模块级 `isReconnecting` 标志：
- `session_start` 自动重连开始时设为 `true`，完成后设为 `false`
- `/chrome-start` 执行前检查该标志，若为 `true` 则等待或拒绝，防止重复连接
